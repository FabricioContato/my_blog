Construtor de Páginas Estáticas em Poucas Palavras

Muitas pessoas não precisam lidar com HTML, CSS e JS para escrever uma página web. Elas podem usar um construtor de páginas estáticas. Esse tipo de software permite que você se concentre em escrever seu conteúdo e usa uma sintaxe de código muito simples, que posteriormente será convertida em código HTML, CSS e JS para você. A desvantagem pode ser que eles não são muito personalizáveis, já que a forma como a conversão é feita é definida pelo desenvolvedor da ferramenta.

Por Que Razão?

Eu tenho escrito e editado meus posts de blog em HTML sem muita automação. Meu processo tem sido:

  • Passo algum tempo no Google Docs fazendo brainstorming.
  • Após terminar a primeira versão, peço ao ChatGPT para verificar a gramática e dar sugestões.
  • Se não estou satisfeito, volto ao brainstorming e peço ao ChatGPT novamente.
  • Quando estou satisfeito com meu texto, edito o arquivo HTML do post.
  • Por fim, git add, git commit e envio meu projeto de blog para meu repositório GitHub, que atualiza meu blog online.

Isso não estava me incomodando até recentemente, quando tive a ideia de ter duas versões de cada página do blog: uma em inglês, para o mundo em geral, e uma em português, para meus colegas brasileiros que convido para ler meu blog. Existem outras soluções para esse problema de idioma, mas falarei mais sobre isso em outro post.

OK, posso escrever um post em um idioma e pedir ao ChatGPT vesão gratuita para traduzi-lo para mim, mas e se eu mudar a estrutura padrão do meu post e depois tiver que passar por cada um manualmente para alterar seu código? Lembre-se de que serão o dobro de arquivos, então o número pode aumentar rapidamente.

Uma solução automatizada que pega arquivos de texto e os transforma no formato do meu blog seria muito útil.

Além disso, seria bom também poder escrever meu texto em documento de texto qualquer e, com um clique, ter a página HTML gerada.

Demonstração

Digamos que eu tenha o seguinte arquivo de texto simples chamado myPostText.txt: 

postFile.html   
#card   
 #center-title Olá Mundo   
 #p Esse é um parágrafo de teste   
 #ul   
  #li Item 1   
  #li Item 2   
 #p tchau tchau   
#card   
 #title Segundo card aqui

A primeira linha define o nome do arquivo de saída. Cada outra linha é composta por uma tag (#tagName) e algumas têm texto após ela.

Cada tag tem um código HTML initial e end associado. Isso cabe a você definir no arquivo chamado tagRules.tr. Abaixo estão minhas definições para esta demonstração.

  • #card initial: <div class="card card-body mb-3"> end: </div>
  • #center-title initial: <h1> end: </h1>
  • #title initial: <h2> end: </h2>
  • #p initial: <p> end: </p>
  • #ul initial: <ul> end: </ul>
  • #li initial: <li> end: </li>

Eu chamo minha aplicação de linha de comando spb (Static Page Builder) assim: 

spb myPostText.txt

Ela carregará todo o conteúdo do arquivo de texto, que o usará para construir a página postFile.html.

Também posso fazer isso para vários arquivos .txt passando uma pasta como primeiro parâmetro: 

spb minhaPasta

Deixarei mais informações sobre como usar meu software em seu arquivo README no GitHub.

Se você chegou até aqui, muito obrigado. Estes slides me deram muito trabalho.

Você pode pensar que falta uma tag body (<body>...</body>) e outras coisas no output final. Isso porque eu abstraí muita coisa que estão envolta do princípio básico de como o programa funciona.

Na verdade, por padrão, a primeira tag a ser adicionada na pilha é a tag #html que você define no arquivo tagRules.tr. No initial e end é onde eu coloco coisas como <header>, <link>, <body>. Sua indentação é definida como zero. Cada objeto criado para tags no arquivo .txt tem sua indentação incrementada em 1, então elas são sempre empilhadas acima da padrão.
